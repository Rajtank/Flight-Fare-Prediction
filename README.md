# Indian Domestic Flight Fare Prediction

This project predicts domestic flight prices within India using a dataset of 11 columns and roughly 11,000 observations. The analysis and exploratory work live in the Jupyter notebook, while the trained model is exposed through a small Streamlit web app.

## Dataset

| Column | Description |
| --- | --- |
| Airline | Carrier the traveler used (e.g. IndiGo, SpiceJet, Air India) |
| Date_of_Journey | Date when the travel occurred |
| Source | Departure city |
| Destination | Arrival city |
| Route | Flight route (e.g. BLR → BOM → DEL) |
| Dep_Time | Take-off time |
| Arrival_Time | Landing time |
| Duration | Total travel time |
| Total_Stops | Number of stops |
| Additional_Info | Extra flight details |
| Price | Flight price (target variable) |

## Project Structure

```
Flight_Fare_Analysis.ipynb   # EDA, cleaning, feature engineering, visualisations
Flight_Fare_Prediction.xlsx  # Raw dataset
train_model.py               # Reproduces preprocessing and trains the Random Forest model
app.py                       # Streamlit UI that loads the trained model and predicts fares
requirements.txt             # Python dependencies
flight_fare_model.pkl        # Generated by train_model.py (not committed)
```

## Data Preprocessing

The notebook performs the following steps:

1. **Null handling** — drops the two rows with missing `Route` / `Total_Stops`.
2. **Date features** — splits `Date_of_Journey` into `Travelled_Date` and `Travelled_month`.
3. **Time features** — extracts hour and minute from `Dep_Time` and `Arrival_Time`.
4. **Duration parsing** — converts strings like `2h 50m` into separate `duration_hour` / `duration_min` columns using regex.
5. **Encoding** — one-hot encodes `Airline`, `Source`, `Destination`; maps `Total_Stops` to integers.
6. **Correlation analysis** — heatmap to inspect feature relationships with `Price`.

## Modeling

The modeling pipeline lives in `train_model.py`. It reuses the notebook's preprocessing and trains a tuned `RandomForestRegressor`:

```python
RandomForestRegressor(
    n_estimators=700,
    min_samples_split=5,
    min_samples_leaf=2,
    max_features="sqrt",
    max_depth=15,
)
```

Performance on an 80/20 split:

| Metric | Score |
| --- | --- |
| Training R² | ~0.88 |
| Testing R²  | ~0.78 |

## Getting Started

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Train the model

```bash
python train_model.py
```

This writes `flight_fare_model.pkl` to the project root.

### 3. Launch the Streamlit app

```bash
streamlit run app.py
```

The app opens at `http://localhost:8501`. Pick airline, source, destination, journey date, departure and arrival times, and number of stops — the app computes flight duration automatically and returns the predicted fare.
